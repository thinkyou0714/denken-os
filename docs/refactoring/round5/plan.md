## 不変条件（4ラウンド維持の保証を壊さない）

1. 生成問題・採点・UI文言・保存データ形式は**不変**。
2. `web/problems.json` は **バイト不変**（`build:problems` 後 `git diff --exit-code` = 0）。
3. 既存テストは無変更で全緑（追加は可）。
4. 公開 export シグネチャを壊さない（追加は可）。
5. カバレッジ閾値（84/76/91/89）維持・`audit:status:strict` 緑。

## 見送り（理由付き・ideas-100.md 参照）
- `checkParamsConsistency` の生成パイプライン強制（**problems.json byte risk: high**・採用draw変化）
- weaknessScore の attempts=0 再重み付け（**挙動変更**＝優先順位が変わる）
- clean.ts / テンプレ数値判定の変更（byte変化＝決定論保証を壊す）
- 観測性フック追加（notify 等）・pick() ラベル（低価値・API面拡大）

## 非問題（誤報・調査の結果バグでないと確認）
実検証で却下した代表例（詳細は ideas-100.md）:
- BACKUP_KEYS の `denken:sound` 欠落 → **存在済み**（backup.ts:42）。誤報
- aggregate 最頻誤答の非決定性 → strict `>` で**最小index決定的**（第3ラウンドN-/T-C5と一致）。誤報
- 数値回答 input の aria 欠落 → **既に `aria-label="数値の答え"`**（practice.ts）。誤報
- 設定 goal/cap が範囲外を localStorage に通す → setter が**従来からクランプ済み**。誤報
- PR テンプレ不在 → **存在済み**。`.gitleaksignore` 行コメント不足 → **冒頭ブロックで説明済み**
