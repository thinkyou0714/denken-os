# 週次コンテンツカレンダー(無理のない頻度)

> 原則: **毎日でなくていい**。続けられる量に絞る(`08-sustainability-kpi.md`)。
> 出題は半自動・予約投稿、人間はリプ/会話に集中。

## 標準ペース(週あたり)

| コンテンツ | 頻度 | 担当 |
|---|---|---|
| 今日の一問(朝出題＋夜解答) | 週3〜4 | 出題=半自動 / 解説=手動 |
| 学習ログ | 週2〜3 | 手動(学習の二毛作) |
| 失敗/あるある共感 | 週1〜2 | 手動 |
| 開発ログ(エンジニア層) | 週1 | 手動 |
| 保存版まとめ | 月1〜2 | バッチ |
| RTされる質(人に教えたくなる) | 週1で必ず | バッチ(Grok=RT20倍狙い。09参照) |
| 総集編 | 月1 | バッチ |
| リプ営業 | 毎日30分 | 手動(最重要) |

> **脱Slop**: 全投稿に一次情報(自分のつまずき/実体験)を1つ入れ、解説は自作・図は手書き。
> 同一文面の連投を避けテンプレは5種以上ローテ。詳細は `../09-anti-slop.md`。

## 週の組み方(例)

```
月  今日の一問(朝→夜)      ／ リプ営業
火  学習ログ                ／ リプ営業
水  今日の一問(朝→夜)      ／ リプ営業
木  あるある共感            ／ リプ営業
金  今日の一問(朝→夜)      ／ リプ営業
土  開発ログ ＋【週末バッチ:翌週分の問題を生成・検算・予約投稿】
日  学習ログ or 休む        ／ リプ営業
```

## バッチ日(週末)チェック

```
□ 翌週の今日の一問 3〜4問を生成
□ 全問を 03 のチェックリスト通過
□ 出典・改題を明記(04)
□ 出題/解答テンプレに流し込み(post-templates.md)
□ 予約投稿セット(最適時間: 朝7時台 / 夜21時台 など)
□ ストックが2週間分あるか確認
```

## 直前期(試験1ヶ月前〜)

- 今日の一問を週1〜2に絞る。自分の勉強を最優先。
- 「ラスト追い込み中」を公開し、失速を物語にする。
