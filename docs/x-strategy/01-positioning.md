# 01. ポジショニング ／ プロフィール文案

## ペルソナ定義

| 項目 | 設定 |
|---|---|
| 立場 | **電験二種に初挑戦する当事者**(合格者・先生ではない) |
| 役割 | 勉強しながら学習アプリ DENKEN-OS を作る開発者 |
| トーン | 等身大・正直・前向き。賢く見せすぎない |
| 制約(共感の核) | 例:文系出身／働きながら／数学苦手 など 1〜2個を明示 |
| 敵(共通の苦しみ) | 分厚い参考書・孤独な独学・進捗が見えない不安 |
| 約束 | 「ゼロから受かるまでの全過程を公開する」 |

> **重要**: 弱い主人公ほど成長曲線がドラマになる。スタート地点の弱さは資産。

---

## 主・副の客層(希薄化を防ぐ)

- **主戦場 = 受験者層**(X勉強垢・電験クラスタ)。発信の8割はここ向け。
- **副 = 個人開発/エンジニア層**(開発ログ)。週1程度。流入経路の二系統目。
- 迷ったら **必ず受験者層を優先**。専門特化がアルゴリズム評価の前提。

---

## プロフィール文案(コピペして調整)

**案A(王道・共感型)**
```
文系SEが電験二種に独学で初挑戦中｜偏差値ゼロから合格まで全公開
📘「今日の一問」毎週配信／勉強が辛すぎて学習アプリ作っちゃいました
一緒に受かりましょう。質問・ツッコミ歓迎🔧 試験まで残り○○日
詳細・アプリ↓
```

**案B(物語強調型)**
```
凡人、電験二種に挑む。｜2026年○月、ノー勉スタート
勉強の記録 × 自作の学習アプリ × 一緒に走る仲間募集
「受かるか分からない人間」の挑戦記録。落ちても続けます
```

**案C(開発者層も意識)**
```
電験二種を独学で攻略中｜Next.js+Supabaseで学習アプリ開発
過去問を"無限に類題化"するエンジンを作ってます
勉強ログ／作問チャレンジ／開発の裏側 ぜんぶ公開
```

### 書き方のルール
- 1行目で「誰が・何に挑み・何をくれるか」を即提示(プロフでフォロー率が2〜3倍変わる)。
- 表示名に「電験二種」を入れる(X内検索＆AIの興味推定に拾われる)。
- リンクは1つに集約(lit.link等)。本文への直貼りはインプ激減なので避ける。

## 固定ポスト案(物語の"第1話")

```
【宣言】電験二種、ノー勉から始めます。

正直、今の実力は模試で○点。受かる保証はゼロ。
でも「凡人がゼロから二種に受かるまで」を全部ここに記録します。

しかも、勉強が辛すぎて学習アプリまで作り始めました（笑）
👉「今日の一問」を一緒に解く
👉つまずきも失敗も隠さない
👉受かったらこのアプリを次の受験者へ

一緒に受かりませんか? まずはこのアカウントをフォローで仲間に。
（アプリ・詳細はプロフのリンクから）
```
- これを固定し、訪問者が最初に見る「物語の入口」にする。
- 画像1枚(ヘッダーと同じ世界観)を添えて滞在時間を稼ぐ。

## ヘッダー画像の文言

```
電験二種まで あと ○○ 日 ／ 凡人の挑戦記録、更新中
```
- 回路図モチーフ＋カウントダウン。「進行中の物語」を一目で伝える。

---

## やらないこと(NG)

- 勉強・電験・開発**以外**の雑多な投稿(専門特化が薄れ評価が下がる)。
- フォロバ目的の相互フォロー(関連性の低いフォロワーは評価を薄める)。
- 賢く見せる・盛る(等身大の苦戦こそ応援される主人公の条件)。
