# 07. 収益ブリッジ ＆ 不合格ヘッジ(根本原因 #2)

## 問題:物語と収益が「合格」という賭けに依存している

- 二種は複数年が普通。**2年落ち続ける期間**の設計が必要。
- 課金は合格後に置くため、**その間ずっと収益ほぼゼロ**は持続性(`08`)を脅かす。
- 「リベンジ編」の一言では弱い。**合格に依存しない価値と収益**を用意する。

---

## 合格非依存の「価値」を作る

合格を待たずに、独立した成果として早期に価値化する。

| 成果物 | 合格しなくても価値がある理由 |
|---|---|
| 学習アプリ DENKEN-OS | ツールとしての有用性は合否と無関係 |
| 一次試験合格 | 二次前でも「一次は受かった」は実績になる |
| 勉強法・ロードマップ | 独学者に再現可能なプロセスを提供 |
| 誤答データ・良問集 | 受験者全体に役立つ資産 |
| コミュニティ | 仲間がいること自体が価値 |

> **物語のリスクヘッジ ＝ 事業のリスクヘッジ**。合格は「最大の山場」だが「唯一の価値」にしない。

---

## 収益ブリッジ(無料 → 課金の階段)

```
フェーズ0(立ち上げ)   完全無料。信頼とフォロワーと母集団を作る
   │  ・「無料で配る」宣言で信頼を爆増
   ▼
フェーズ1(早期の小さな売上)  アフィリエイト（参考書・電卓レビュー）
   │  ・在庫リスクゼロ。検索需要も大きい
   ▼
フェーズ2(検証)  note 有料記事（数百円）で需要と単価を低リスク検証
   │  例:「二種 独学ロードマップ完全版」「一次合格までの記録」
   ▼
フェーズ3(プロダクト課金)  アプリ フリーミアム
   │  ・基礎無料 / AI生成・弱点診断・記録・解説 を有料
   ▼
フェーズ4(ストック収益)  会員制コミュニティ（月額）
   │  ・作業会・添削・限定解説。長期戦を支える
   ▼
フェーズ5(合格後)  実績で値付けを正当化 → 値上げ・法人/スクール展開
```

- **課金は合格 or 一定規模の後**が原則だが、フェーズ1〜2の**橋渡し収益**で
  ゼロ円期間の持続性リスクを緩和する。

---

## フリーミアムの線引き(カニバリ回避)

| 無料(発信＆アプリ無料枠) | 有料(アプリ) |
|---|---|
| 「今日の一問」1日1問 | 類題を**無限に**演習 |
| 答えと要点 | 深い解説・解法動画 |
| 固定問題 | **弱点に合わせた適応出題** |
| 記録のシェア画像 | **学習記録・進捗管理の蓄積** |

- タダ出しは「サンプル/予告編」、アプリは「本編(量・適応・記録・解説)」。
- 「なぜ課金が要るか」を発信で誠実に語り、納得を作る。

## 不合格・長期戦の物語ヘッジ

事前に台本を用意し、最大リスクを物語に転換する。

```
■ もし不合格だったら
  1. 隠さず公開:「落ちました。何が足りなかったか全部分析します」
     → 失敗の全記録は成功談だらけの界隈で希少価値。むしろ伸びる
  2. 「リベンジ編 開幕」として第2シーズン化(連載は続く)
  3. 一次合格・アプリ・コミュニティを「今年得たもの」として可視化
  4. 同じく落ちた仲間の出戻りを歓迎 → コミュニティが強くなる

■ もし合格したら(最大の山場)
  1. 「ゼロから自作アプリで受かった」全記録の大型スレを投下
     → 権威・信頼・アプリ価値を一気に確立(全伏線回収)
  2. 値付けを正当化し、課金フェーズへ
  3. 「次世代を導く」師匠役へ転換 → 物語もアプリも終わらない
```
- どちらに転んでも**物語は続く**設計。これが合格依存リスクの本質的ヘッジ。
