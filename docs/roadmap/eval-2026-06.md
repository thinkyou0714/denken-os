## 評価サマリ

現 main は pre-alpha ながら作り込みが高い：決定論 solver（答案を LLM に出させずコードで算出）、
schema/exam↔subject 整合、RLS DDL、固定シードの不変条件スイープ、CI 品質ゲート、governance hardening。
本評価では、その設計の核心（反ハルシネーション・SR 診断・永続化 parity）に残る根本原因バグを検出・修正した。

## 実装済み（本リリース）

### 検証済み root-cause バグ（敵対的検証で isReal=true）

| ID | バグ | 修正 |
|---|---|---|
| F1/DI-1 | 反ハルシネーションゲート `narrationMatchesAnswer` が「解説中のどこかの数値」一致で合格（最終値非アンカー）。cos=1.0 変圧器では答え=p が中間式に必ず現れ、最終値を誤っても素通り（設計の核心保証が無効化） | 結論の最終値（`=` 以降が数値+単位だけのステップの最後）にアンカー。`validate.test.ts` に偽陽性負テスト追加 |
| SCHED-2 | 弱点診断が `dueMs=最終解答時刻` を使い、overdue=放置日数に化け、最近やった topic を不利・習得済の放置 topic を弱点に誤判定（SR セマンティクス逆転） | スケジューラの実 due を渡せるよう拡張（`app.ts` が `allReviews` から供給）。`diagnosis.test.ts` に未来due/過去due・習得済 vs 低正答率の順位テスト |

### 補強テスト

- property-based シード探索（fast-check）でエンジン不変条件 + generate↔generateFrom 決定論を乱数シードで検証。
- ajv⇔zod 差分テスト（fast-check 500 ケース）で schema ドリフトを挙動一致で根絶。

検証: `npm run verify` 緑、205 tests pass、coverage 89.39/81.01/95.08/92.54（閾値 85/76/92/89）。

## ロードマップ（未実装。評価で検出した medium/low + 構造改善）

### データ整合（評価 DI-3..6）
- **DI-3** `realistic_range` を生成器の実 draw 域と一致させる（保存値が広すぎ、メタ整合契約が緩い）。
- **DI-4** 離散パラメータの列挙制約（誘導機の極数=偶数、商用周波数∈{50,60}）を validate に追加。
- **DI-5** `human_checked`/`supervisor_checked` を provenance 付き（checked_by/at/method）に拡張し、validated 昇格の根拠を必須化。
- **DI-6** `past_exam_*` の citation 書式（年度・区分・科目）を検証。

### 構造・運用
- **統一 Scheduler の web 採用**: `FsrsReviewScheduler`（本リリースで導入）を deck 設定で SM-2 と切替（現状 web は SM-2 固定）。
- **CI-1 週次 cron 監査**: pinned 依存の新規 CVE を PR 非依存で検知する schedule workflow。
- **IndexedDB 移行 / Supabase magic-link auth / leech 検出・lapse queue / 2回目誤答 pedagogy / 前提科目順**（原評価 Tier2/3）。

### 保守メモ
- zod v4 へ移行済（`z.toJSONSchema` 利用可）。JSON Schema codegen を入れるなら superRefine 条件は overlay 合成 + 差分テストで担保（直列化不可のため）。
