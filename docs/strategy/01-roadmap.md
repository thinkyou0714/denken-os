# X発信 マスターロードマップ

`M = 開始からの月数`。受験年に合わせて試験日を当てはめる。
電験二種の目安: **一次=8月下旬／二次=11月中旬／一次結果=10月／二次結果=翌2月**。

進行は「期間」でなく「KPIゲート」で進む（条件を満たしたら次へ。焦らない）。

---

## フェーズ別の詳細

### P0 種まき（M0 / 最初の1週間）
- ゴール: 物語の入口を完成させる
- □ プロフ・固定ポスト・ヘッダー設定 / □「ノー勉宣言」を固定 / □ リンク集約 / □ #電験二種 を20人フォロー
- KPI: 設定完了（フォロワー数は見ない）
- → 進む条件: 固定ポストが投稿済み

### P1 記録（M1–M2）
- ゴール: 当事者として認知され、同期クラスタを作る
- 毎週: 学習ログ週3／あるある週1／今日の一問は週1で試運転
- 毎日: ★リプ営業30分（大手へ補足リプ＋同期5–10人と相互応援）
- アプリ: 自分用に開発（ドッグフーディング、非公開）
- KPI目安: フォロワー 300–500 ／ 同期 5–10人 ／ 毎日投稿の習慣化
- → 進む条件: 反応してくれる「同期」が5人以上

### P2 道具（M2–M4）
- ゴール: アプリを"副産物"として公開し、今日の一問を主力化
- 毎週: 今日の一問 週3–4（朝出題→夜解答）／開発ログ週1／学習ログ週2
- イベント: アプリβテスター「先着50名」募集（M3）
- アプリ: 検算パイプライン稼働 → βリリース（無料）
- コンプラ: 全問チェックリスト通過＋出典明記（事故ゼロ厳守）
- KPI目安: フォロワー 1,000–2,000 ／ βテスター 50 ／ 保存が付き始める
- → 進む条件: 今日の一問が安定して保存・リプを獲得

### P3 共闘（M4–M7）
- ゴール: コミュニティ＝堀を作り、拡散ループを回す
- 毎週: 今日の一問 週4／週次もくもく会／作問チャレンジ
- イベント: Discord/オプチャ開設（M4）／月1「保存版まとめ」投下
- アプリ: 学習記録シェア画像／招待機能でバイラルループ
- 収益: アフィリ開始＋note有料記事で需要検証（橋渡し収益）
- KPI目安: フォロワー 3,000–5,000 ／ コミュニティ 100人＋ ／ 紹介流入発生
- → 進む条件: コミュニティが自走（メンバー同士のQ&A発生）

### P4 信頼／直前（M7–試験）
- ゴール: 権威ギャップを埋め、一次試験に挑む
- 毎週: 直前期は今日の一問を週1–2に絞り、自分の勉強を最優先
- 仕込み: 合格者を監修者に確保／合格事例(他者)を1件作り込む
- 発信: 「ラスト追い込み中」を公開／模試スコア公開
- ［8月］一次試験 受験 → リアルタイム実況スレで当日トラフィック総取り
- KPI目安: フォロワー 5,000–10,000 ／ 監修者確保 ／ 一次合格
- → 分岐: 合格 or 不合格 どちらでも物語は続く（07のヘッジ発動）

### P5 成り上がり（試験後〜）
- ゴール: 合格で全伏線回収 → マネタイズ → 次世代を導く
- ［合格］「ゼロから自作アプリで受かった」全記録の大型スレ＝最大の山場
- 収益: フリーミアム課金開始 → 会員制 → 実績で値上げ → 法人/スクール
- 役割転換: 主人公 → 師匠（合格者が次の受験者を導く循環）
- ［不合格］「リベンジ編 開幕」第2シーズン化＋一次合格/アプリを価値化

## 最初の90日 ズーム

```
 W1  基盤構築。プロフ/固定/ヘッダー。リプ営業スタート
 W2  学習ログ毎日。大手3アカウントに毎日リプ。同期探し
 W3  今日の一問 試運転（週1）。反応を見る
 W4  ★1ヶ月レビュー：保存/リプが付く型を特定
 ───────────────────────────────────────────────
 W5  今日の一問 週2へ。検算パイプライン構築開始
 W6  あるある/失敗共有で共感バズを狙う
 W7  開発ログ投入（エンジニア層へ二系統目の流入）
 W8  ★初の「保存版まとめ」投下。初速30分に張り付く
 ───────────────────────────────────────────────
 W9  アプリβ告知。「先着50名」待機リスト募集
 W10 βリリース。今日の一問 週3–4へ本格化
 W11 βテスターの声をUGCで発信。改善ループ
 W12 ★90日レビュー：KPI3つで当たり型に資源集中 → P3準備
```

## 週次オペレーション（繰り返すエンジン）

```
 月 今日の一問(朝出題→夜解答)  ＋ リプ営業30分
 火 学習ログ                   ＋ リプ営業30分
 水 今日の一問(朝→夜)          ＋ リプ営業30分
 木 あるある共感               ＋ リプ営業30分
 金 今日の一問(朝→夜)          ＋ リプ営業30分
 土 開発ログ ＋【週末バッチ：翌週分を生成・検算・予約】
 日 学習ログ or 休む           ＋ リプ営業30分
   ↑ 出題は半自動・予約／人間はリプと解説に集中（持続性の要）
```

## KPIゲート（次へ進む鍵）

```
 P0 → P1 :  固定ポスト完成
 P1 → P2 :  同期5人＋／毎日投稿が習慣化
 P2 → P3 :  今日の一問が安定して保存・リプ獲得／βテスター50
 P3 → P4 :  コミュニティ自走（メンバー同士のQ&A）／紹介流入発生
 P4 → P5 :  一次試験 受験（合否どちらでも物語続行）
 P5      :  合格→マネタイズ全開／不合格→リベンジ編

 ★追う指標は常に3つ:  保存数 ／ リプ往復 ／ フォロー転換
   （フォロワー数・インプはバニティ指標。主指標にしない）
```

> KPIの数値は目安。`docs/x-strategy/08-sustainability-kpi.md` の週次レビューで検証して調整する。
