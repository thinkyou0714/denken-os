# 戦略オーバービュー ──「凡人が成り上がる」サクセスストーリー

## コンセプト

電験界隈の発信は「すでに合格した権威者」による過去問解説がほぼ独占している
（Taira、電験王、カフェジカ、あずまや 等）。そこで本プロジェクトは逆を行く:

> **これから初めて電験二種を受ける当事者**が、**学習アプリを作りながら**、
> **一緒に走る仲間を集める** ── という空白地帯のポジション。

「自分も初めて電験二種を受けます」というキャラクターで、
「勉強するにあたって学習アプリを個人開発しちゃいました、試しに一緒に勉強しませんか？」
という流れ。これは "build/learn in public" の路線。

### なぜ筋が良いか（差別化の根拠）
1. **差別化** — 解説の質で大手に勝つのは無理。だが「当事者の挑戦プロセス」「ツール」「正直さ」は競合がいない。
2. **二重コンテンツ** — 学習発信（受験者層）＋開発発信（個人開発/エンジニア層）でネタ2倍・流入2系統。
3. **物語性** — 「受かるか分からない人間が挑む」は stakes があり、合格者の解説にはない"続きが気になる"力。
4. **アプリが follow 理由になる** — ただの勉強垢でなく「あの人が作ってるやつ」という固有のフック。

### 最大の論点（弱点）
**未合格者の権威ギャップ** — 買い手は「合格者」を信頼する。これを透明性・データ・監修・誠実さで埋める。
詳細な評価は `02-evaluation.md`。

## 成り上がりの階段（サクセスストーリーの弧）

```
                                                    ┌────────────────────┐
                                                    │ Lv.5  成り上がり    │
                                                    │ 合格→事業化         │
                                            ┌───────┴────────────────────┘
                                            │ Lv.4  信頼/実績
                                            │ 合格事例・データ
                                    ┌───────┴────────────┐
                                    │ Lv.3  共闘          │
                                    │ コミュニティ&拡散   │
                            ┌───────┴────────────┐
                            │ Lv.2  道具          │
                            │ アプリを"副産物"公開 │
                    ┌───────┴────────────┐
                    │ Lv.1  記録          │
                    │ 当事者の発信開始     │
            ┌───────┴────────────┐
            │ Lv.0  凡人          │
            │ ノー勉宣言・ゼロ地点 │
            └─────────────────────┘
```

| Lv | 自分の立場 | 発信 | アプリ | 物語の見せ場 |
|---|---|---|---|---|
| 0 凡人 | ただの初学者 | ノー勉宣言・実力を晒す | 構想だけ予告 | 「弱い主人公」の起点＝共感 |
| 1 記録 | 挑戦者 | 日次進捗/つまずき/失敗共有 | 自分用に開発(ドッグフード) | 毎日の前進・サボり→復活 |
| 2 道具 | 作る人 | 開発ログ/機能デモ | 無料公開・βテスト・弱点診断 | 「辛くて作っちゃった」＝二重コンテンツ |
| 3 共闘 | 旗振り役 | 共通タグ・合同勉強会 | シェア機能/招待で拡散 | 仲間が増える・一緒に走る |
| 4 信頼 | 半・先輩 | 合格事例・データ・監修 | 利用実績/合格率=社会的証明 | 仲間の合格＝権威ギャップを他者の成功で補う |
| 5 成り上り | 合格者兼オーナー | 合格体験の全公開→次世代へ | フリーミアム課金・事業化 | 自分の合格で全伏線回収＝サクセス確定 |

## 設計意図（物語の肝）

- **「弱い主人公」から始める** — 実力を晒すほど成長曲線がドラマになる。完璧な先生でなく応援したくなる凡人。
- **アプリ＝物語の中の「武器」** — 「勉強が辛くて作っちゃった」という必然で登場させ宣伝臭を消す。
- **最大の山場は合格(Lv.4→5)** — 権威ギャップが一気に解消し、商品の信頼とマネタイズが正当化される。だから課金は合格まで引っ張る。
- **不合格でも物語は死なない** — 「リベンジ編」に切替えられる構造。二種は複数年が普通なので長期連載として設計。
