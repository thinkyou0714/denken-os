# 戦略の評価（率直なスコアリングと診断）

「全部実装した前提」での率直な評価。戦略"設計"としては一流だが、"成功確率"込みでは満点ではない。

## 採点結果: 総合 82 / 100

| 評価カテゴリ | 配点 | 得点 | 講評 |
|---|---|---|---|
| ① ポジショニング・差別化 | 15 | 13 | 権威者だらけの市場で「当事者×開発者」は白地。ほぼ満点級 |
| ② Xアルゴリズム適合 | 12 | 11 | 会話/保存/滞在/初速/リンク回避を全網羅 |

**2つの軸:**
- 戦略"設計"の完成度 → 約90点（計画書としてはトップクラス）
- "成功確率"込みの総合 → 82点（計画では消せない構造リスクで減点）

## なぜ100点にならないか（構造的な天井 = 残り18点）

実装でも気合でも消せない5つの根本原因。

### 1. 一人の処理能力を超える前提（最大の減点・⑧）
毎日: 生成→検算→人の目→解説→出題→リプ→解答→自分の勉強→アプリ開発。
バッチ/テンプレ化しても1人の容量を超える。「全部やろうとして潰れる」のが最大の破綻要因。
→ **スコープを削る**（毎日→週3、出題は自動・解説だけ手動）、共同運営者/監修者で分散。

### 2. 物語と収益が「合格」という賭けに依存（⑨⑦）
全アークと商品の信頼が「いつか合格」に依存。2年落ち続ける期間の設計が弱く、その間収益ほぼゼロ。
→ **合格非依存の価値**（一次合格・アプリ・勉強法）と**収益の橋**（アフィリ/note）を早めに。

### 3. 差別化が"数ヶ月の堀"でしかない（⑥）
「AI無限生成×今日の一問」が当たると、配信力で勝る大手が真似して追い抜ける。
→ **模倣されにくい資産を堀に**: コミュニティ・合格者ネットワーク・誤答データ・物語。

### 4. 市場の天井が低い（①）
電験二種はニッチ。完全制覇しても総リーチに上限。二刀流は客層を希釈。
→ **三種を入口に母数を取る**、将来は他資格へ横展開。

### 5. 品質は99%でも"1問の事故"が効く（④）
検算しても、たまに通り抜ける誤答が1つバズれば信頼が傷つく。確率ゼロにはできない。
→ **「誤りは出る前提」で訂正運用を最速化**し、訂正の誠実さをブランドにする。

## 「今日の一問」前提での10の問題点と根本原因

「係数いじりで無限生成」を主力にする場合の潰すべき穴。詳細な対策は `ideas/04-100point-fixes.md`。

| 問題点 | 根本原因 | 改善 |
|---|---|---|
| ① 生成問題が間違い/答えが汚い/非現実的 | 検証プロセス無し＋未合格で判定不能 | 検算パイプライン化 |
| ② 過去問改題が著作権違反になりうる | 利用ルール未把握 | 出典＋改題を必ず明記 |
| ③「どうかな?」連発が素人臭/エンゲベイト | 謙虚キャラと商品の権威が衝突 | "検証企画"として設計化 |
| ④ 出題と解答が同じ投稿で会話ループが死ぬ | クイズのエンゲージ機構未活用 | 朝出題→夜解答の二段設計 |
| ⑤「今日の一問」は模倣容易でmoatが無い | コモディティ化 | 無限生成＋適応＋記述をmoat化 |
| ⑥ 良問を全部タダ出し→課金理由が消える | 無料/有料の線引き不在 | 発信=予告編、アプリ=本編 |
| ⑦ クイズ化しやすい一次計算に寄り差別化を逃す | 型が易しい方へ流れる | 二次(記述)を主戦場に |
| ⑧ 毎日良問＋解説で燃え尽き | コンテンツトレッドミル | バッチ＋テンプレ＋生成エンジン |
| ⑨ 単発問題が流れて消える | アーカイブ性ゼロ | 連番＋総集編＋アプリが保管庫 |
| ⑩「AI生成問題」が質を疑われる | 2026のAIスロップ懸念 | human-in-the-loopを明示 |

## 100点に近づける「あと3手」（82→90台）

1. **スコープを削る勇気**（⑧→+3） — 出題は自動化、人間は解説と会話に集中。続けられる設計が9割。
2. **合格非依存の価値を作る**（⑦⑨→+4） — 一次合格・アプリ・勉強法を独立した成果として早期に価値化。
3. **堀を技術から関係性へ移す**（⑥→+2） — コミュニティ・合格者ネットワーク・物語を最優先で育てる。

### 総評
**「計画」としては90点、「これで成功するか」としては82点。** 残りは現実というテストでしか埋まらない。
弱点は中身ではなく「全部やろうとして潰れること」と「合格という単一障害点」の2つにほぼ集約される。
そこを「削る/分散する/ヘッジする」で設計し直せば、地に足のついた強い計画。
