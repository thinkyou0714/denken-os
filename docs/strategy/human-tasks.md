# 人間タスク チェックリスト（DENKEN-OS）

このセッションで設計した戦略・運用・自動化を実現するうえで、**人間(本人)にしかできない／
自分で手続きする必要があるタスク**を、もれなく・フェーズ順・時間制約つきで洗い出したもの。
ドキュメントやコード、AIでは肩代わりできないものに絞っている。

凡例: ⏰=締切あり / 🔁=日々の継続 / 🤝=委任検討可 / ⚖️=法務・事務

## 根本原因対策:「処理能力超過」を防ぐ ── 委任/捨てる候補
最大リスクは「全部やろうとして潰れる」こと（`02-evaluation.md`）。最初から手放す前提で設計する。

- 🤝 **委任候補**: 監修者へ品質チェック／共同運営者へ運用分散／コミュニティへ質問対応／会計ソフト・税理士へ事務
- ✂️ **後回し/捨てる候補（立ち上げ期）**: 商標出願・法人化・アプリ配信・複数SNS展開・課金まわり一式
  → これらは「合格 or 一定規模の後」で十分。立ち上げ期は **勉強＋X＋一次情報** に絞る。
- 原則: **人間=勉強・一次情報・会話/関係性・最終承認・温度**／**AI=骨格生成・検算・予約・集計・通知・下書き**
