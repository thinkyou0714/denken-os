# 電験二種 試験構造と学習設計

> 「この学習アプリで一発合格する」ための前提地図。試験の科目・配点・合格基準を明確にし、
> 学習科学に基づく学習法と、本アプリ各機能の対応を示す。
> 出典は末尾。試験要項は年度で変わるため、出願前に必ず[電気技術者試験センター](https://www.shiken.or.jp/)で確認すること。

## 1. 試験の全体像

第二種電気主任技術者試験（電験二種）は **一次試験（4科目）→ 二次試験（2科目）** の2段階。

| 区分 | 科目 | 形式 | 時間(目安) | 合格基準(目安) |
|---|---|---|---|---|
| 一次 | 理論 | マークシート(択一) | 90分 | 各科目おおむね60%以上 |
| 一次 | 電力 | マークシート(択一) | 90分 | 〃 |
| 一次 | 機械 | マークシート(択一) | 90分 | 〃 |
| 一次 | 法規 | マークシート(択一) | 65分 | 〃 |
| 二次 | 電力・管理 | 記述（計算＋論説、6問中4問選択） | 120分 | 2科目合計でおおむね60%以上(108/180目安) |
| 二次 | 機械・制御 | 記述（計算中心、4問中2問選択） | 60分 | 〃 |

- **科目合格制度（一次）**: 一次は科目別合格が可能で、合格科目は**当年含め2年間有効**。翌年は残り科目のみ受験できる。
- **一次免除**: 一次合格者が二次に不合格でも、**翌年は一次が免除**され二次から受験できる。
- **配点（二次の目安）**: 各問30点。電力・管理＝30×4＝120点満点、機械・制御＝30×2＝60点満点、合計180点。**108点以上（≒60%）で合格**。

> 一次は択一だが**公式の導出を問う問題**も多く、丸暗記より理解が効く。
> 二次は記述式で選択肢のヒントが無く、**計算量が三種の比でない**。
> 機械・制御は論説の出題率が低く**ほぼ計算問題**、電力・管理は計算/論説の両系が出る。

## 2. 合格の急所（なぜ落ちるか）

1. **二次対策の出遅れ**: 一次に集中して二次が間に合わない。**一次の段階から二次を見据える**のが定石（二次に一次の倍以上の時間を割く心構え）。
2. **記述に慣れていない**: 暗記だけでは解けず、**導出過程を書く訓練**が要る。
3. **計算速度不足**: 二次は時間との戦い。電卓技能と典型解法の即応性が合否を分ける。
4. **演習量不足**: 過去問・類題の反復が足りない。**平成7年(新制度)以降の過去問**反復が王道。
5. **弱点の放置**: 苦手科目が一次の足切り（各60%）に引っかかる。

## 3. 学習科学に基づく学習法（本アプリの設計原理）

エビングハウスの忘却曲線・分散効果・想起練習効果に基づく、再現性のある学習プロセス。

| 原理 | 内容 | 本アプリの実装 |
|---|---|---|
| 想起練習(active recall) | 読み返すより**思い出す**方が定着する（再読4回より、1回読+3回テストが1週間後+50%） | 解答前に必ず自力で答えを出す→即フィードバック |
| 分散学習(spaced repetition) | 詰め込みより毎日10〜20分の分散。**ほぼ忘れた頃に復習** | **FSRS**で「忘れる直前」に自動復習 |
| FSRS | 安定度・難易度・想起可能性を予測する最新アルゴリズム。同保持率を20〜30%少ない復習で達成 | 4段階評価で記憶状態を更新・目標保持率設定 |
| インターリービング | 単一解法の反復より**混在**で「どの解法か」を判断する力がつく（初学はブロック練習が有効） | 科目ドリル(ブロック)↔苦手優先(混在)を切替 |
| 誤答からの学習 | 間違いの重点復習が伸びを生む | **間違いノート**で誤答problemを再演習 |
| 本番想定の練習(test-like) | 本番に近い条件での演習が転移を高める | **模試**(時間制限・合格ライン・科目内訳) |

### 推奨カリキュラム（フェーズ）

1. **基礎固め**: 科目ドリルで各科目の典型問題を一巡（公式集を併用し導出を理解）。
2. **弱点克服**: 苦手優先モードで弱点論点を集中、間違いノートを潰す。
3. **二次記述**: descriptive問題で導出過程を書く訓練（早期から並行）。
4. **直前期**: 模試で時間配分と本番耐性、復習キューで記憶を維持。

> 目安配分: **二次に一次の2倍以上**の学習時間。毎日「新規＋復習」を回し、復習を優先。

## 4. アプリ機能と試験対策の対応表

| 試験対策の要請 | 本アプリ機能 |
|---|---|
| 各科目の演習量確保 | 29テンプレからの自動生成（220問〜・無限生成） |
| 二次=記述の作法 | descriptive問題＋模範解答→4段階自己採点 |
| 忘却対策 | FSRS復習スケジュール＋復習キュー |
| 弱点の特定と克服 | 弱点診断・科目別到達度・間違いノート |
| 本番耐性・時間配分 | 模試（時間制限・60%判定・科目内訳） |
| 公式の理解 | 公式集（導出の足がかり）＋解説の数式整形 |
| 計画と継続 | 試験カウントダウン・日次目標・連続日数 |

## 5. 現状の限界と次の一手

- 生成問題は**コード検算済みだが未監修（draft）**。本番教材化には**合格者/有資格者の監修**が必須（`docs/strategy/human-tasks.md`）。
- 一次フル模試（科目別足切り再現）・論説テンプレ・採点ルーブリックは**ロードマップ**（`ideas/06-exam-mastery-100.md`）。
- 過去問の取り込みは**出典規約遵守**のうえで段階導入（`lib/ingest`・原典/生成を分離）。
