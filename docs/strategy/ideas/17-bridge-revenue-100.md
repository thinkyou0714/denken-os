# 17. 橋渡し収益（C案）深堀り100 — アフィリエイト・note・導線・ガバナンス

> 戦略 [`../../x-strategy/07-monetization-failure-hedge.md`](../../x-strategy/07-monetization-failure-hedge.md) の
> フェーズ1（アフィリエイト）〜2（note）を実装する100案カタログ。
> 4系統×25案を並列調査で抽出し、次の4値でトリアージした。
>
> - **implement-now**: 外部アカウント不要。今アプリ/リポジトリに入れる（実装済み）
> - **config-later**: コードは今実装。ユーザーがID/URLを設定した時点で発火（実装済み・既定は非表示=fail-open）
> - **ops-only**: コード不要のアプリ外運用（human-tasks へ）
> - **defer**: リスク大 or 後フェーズで再評価
>
> 実装の設計原則はフリーミアム基盤と同一: **config駆動・未設定なら完全非表示・既存挙動不変**。
> さらに本ラウンドで **非侵襲原則を機構化**（頻度制御・オプトアウト・学習中ガード・開示バッジ）。

## 非侵襲原則（本ラウンドで機構化した憲法）

1. **未設定なら存在しない**: 全導線は config 空文字で完全非表示（フリーミアム基盤と同一のfail-open）。
2. **1日1件**: 能動的な収益ナッジはアプリ全体で1日合計1件まで（`web/src/bridge.ts` の頻度制御）。
3. **学習中は出さない**: 出題〜採点完了の間はナッジ禁止（集中ガード）。
4. **消せる**: 設定タブのオプトアウトで全ナッジ停止（機能ゲート=ペイウォール自体は対象外）。
5. **開示は構造で**: アフィリリンクは開示バッジ経由でしか描画されない。
6. **計測は端末内のみ**: 表示/クリックは整数カウントを localStorage に保存。外部送信なし・書き出しは手動。
