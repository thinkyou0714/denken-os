# 10 — AI質問チャット（電験なんでも回答ロボット）深掘り100

> 「電験関連の質問になんでも回答できるAIチャット」を、DENKEN-OS の制約
> （静的PWA・バックエンド無し・オフライン堅持・ハルシネーション根本対策）の中で成立させる。
> UI設計・アーキテクチャ・品質保証・コスト・安全の100アイデアを深掘りし、実装可能なものから織り込む。

## 根本診断（「チャットボットを置くだけ」では破綻する理由）

| # | 問題点 | 根本原因 | 対策方針 |
|---|---|---|---|
| C1 | サーバが無いのにチャットだけサーバ前提になりがち | GitHub Pages の静的配信（バックエンド無し）という前提の見落とし | 2層構成: ①内蔵ナレッジ（オフライン・決定論）を既定、②BYOK でブラウザ→Anthropic API 直結 |
| C2 | LLM は嘘を流暢につく（特に法規の数値・条文） | パラメトリック記憶に正誤の概念がない | 検証済みナレッジへの接地（RAG）＋出典強制＋「わからない」を言える設計＋法規ディスクレーマー自動付与 |
| C3 | APIキーの預かりはリスクの塊 | 鍵をサーバに置くと漏えい・コスト爆発の単一障害点になる | 鍵は預からない（BYOK・localStorage のみ・送信先は Anthropic のみ・共有端末警告） |
| C4 | コストが青天井になり得る | ストリーミング・履歴全送信・上限なし max_tokens | 履歴は直近12ターン・max_tokens 1024・モデル選択（Haiku/Sonnet）・多重送信ガード |
| C5 | 法規・制度は改正で正解が変わる | 知識の鮮度問題（カットオフ） | 法規/制度カテゴリの回答に「最新の条文・公式発表を確認」を構造的に必ず付与 |
| C6 | 「なんでも回答」は範囲外質問の誤答誘発装置 | スコープ未定義 | 範囲を電験に明示（システムプロンプト＋ローカルは検索閾値で正直にフォールバック） |

**凡例**: ✅ 本イテレーションで実装 ／ 🔜 次イテレーション ／ 👤 人手・運用・要素材

## A. アーキテクチャ（1–10）
1. ✅ 2層構成: 内蔵ナレッジ（オフライン既定）＋ BYOK Claude API（任意強化）。
2. ✅ ローカル層は curated ナレッジベース（`lib/chat/knowledge.ts`、出典必須）から引用のみで回答＝構造的にハルシネーション不能。
3. ✅ API層はローカル検索結果をシステムプロンプトに注入（RAG）して接地。
4. ✅ チャットロジックは `lib/chat/`（純ロジック・テスト可能）と `web/src/chat.ts`（DOM/IO）に分離。
5. ✅ SSE パースを純関数化（`drainSseBuffer`/`extractTextDelta`）してユニットテスト。
6. ✅ API障害時はローカル回答へ自動フォールバック（沈黙させない）。
7. 🔜 Web Worker 化で UI スレッドから検索を分離（ナレッジ増大時）。
8. 🔜 Ollama 等ローカルLLM への接続オプション（README ビジョンの オフラインQA）。
9. 🔜 Supabase Edge Functions 経由のプロキシ（M2 で認証導入後、鍵を端末から消す）。
10. 👤 商用提供時のAPIコスト負担モデル（無料枠/課金）の設計。

## B. ハルシネーション対策・グラウンディング（11–20）
11. ✅ 全ナレッジエントリに出典（法規は条文名）を必須化し、回答末尾に表示。
12. ✅ システムプロンプトで「不確かなら不確かと言う」「知らないことは知らない」を強制。
13. ✅ ローカル検索が閾値未満なら知ったかぶりせず「見つからない」と正直に案内。
14. ✅ 法規・制度カテゴリの回答に改正注意ディスクレーマーを自動付与（プロンプト任せにしない）。
15. ✅ 数値計算は途中式＋単位の明記をシステムプロンプトで強制（検算可能な形）。
16. ✅ ナレッジの数値・公式は検算済みテンプレ（`lib/engine/templates`）・公式集と整合させる。
17. ✅ プロンプトインジェクション対策（ユーザー文中の指示上書きに従わない指示＋ローカル層は構造的に無効）。
18. 🔜 回答中の数値を正規表現で抽出し、エンジンのソルバで照合（narrate.ts と同じ検算ゲート）。
19. 🔜 「この回答は役に立った/間違っている」フィードバックボタン→誤答コーパス化。
20. 👤 ナレッジベースの有資格者監修（human-tasks に追加）。

## C. ナレッジベースとローカル検索（21–30）
21. ✅ 60エントリの curated ナレッジ（理論14・電力12・機械12・法規8・制度8・学習法6）。
22. ✅ エイリアス辞書（%Z⇔%インピーダンス、三種⇔電験3種 等）で表記ゆれを吸収。
23. ✅ NFKC 正規化（全角％→%・大文字小文字）＋約物除去で日本語クエリを頑健に。
24. ✅ 文字バイグラム Dice 係数＋用語包含一致のハイブリッドスコア（形態素解析・埋め込み不要＝オフライン決定論）。
25. ✅ `related` グラフで関連トピックを提示（知識の芋づる学習）。
26. ✅ ナレッジ整合性テスト（id一意・related参照実在・出典必須・法規は条文名）。
27. 🔜 問題テンプレ50種の解説をナレッジに自動取込（build時生成）。
28. 🔜 公式集（formulas.ts）とナレッジの単一ソース化（重複定義の解消）。
29. 🔜 エントリ数を200+へ拡充（過去問頻出論点を網羅）。
30. 🔜 軽量ベクトル検索（事前計算した埋め込みを静的JSON配布）への進化。

## 実装サマリ（本イテレーション）

- `lib/chat/` — types / knowledge（60エントリ・出典必須）/ retrieve（バイグラム検索）/ answer（ローカル回答合成）/ prompt（RAG＋ガードレール）
- `web/src/chat.ts` — 履歴永続化・SSEストリーミングクライアント（BYOK）
- `web/src/app.ts` — 質問タブ（吹き出し・チップ・ストリーミング・モード表示）
- `web/src/settings.ts` — APIキー・モデル設定
- `web/index.html` — チャットUIのスタイル（デザイントークン準拠）／ `web/sw.js` v11
- テスト: `tests/chat/`（retrieve/answer/prompt）・`tests/web/chat.test.ts`
