# 実装指示 10: 誤り訂正モニタ（指摘検知→訂正フロー）

> ステータス: 指示のみ（未実装）。

## 0. ゴール
「今日の一問」への誤り指摘リプを検知して**人間にフラグ**を上げ、
確認後に該当問題を `retracted` にし、訂正テンプレ投稿を下書きする。
「1問の事故」を最速で潰す（`03-quality-pipeline.md` の訂正SOPの自動化）。

## 1. 核心の設計判断 ★（自動検知・人間が承認）
- 検知は自動、**訂正・撤回の実行は人間が承認**（任意のリプで自動撤回すると荒らしに悪用される）。
- 誤りは「出る前提」。**検知から訂正までの時間を最短化**することが信頼を守る。
- 訂正は隠さず経緯を残す＋指摘者をクレジット（誠実運用をブランドにする）。

## 2. スタック
- TypeScript + Node。X API（メンション/リプ取得）。LLM で「誤り指摘っぽさ」を分類（補助）。

## 3. 機能要件
- 自分の出題への新規リプを監視し、「誤り指摘の可能性」を分類してアラート（人間へ）
- 人間が「誤り」と判定 → problem.status=retracted、アプリ側データも修正
- 訂正リプ下書き生成（元投稿にぶら下げ／指摘者クレジット／消さない）
- 月次の「訂正ログ」まとめ下書きを生成

## 4. 受け入れ条件
- [ ] 誤り指摘候補が人間に通知される（30分以内目標の運用前提）
- [ ] 自動で撤回・訂正投稿しない（必ず人間承認）
- [ ] 撤回時に status=retracted となりアプリと整合
- [ ] 訂正下書きに指摘者クレジット欄がある

## 5. やらないこと
- リプだけで自動撤回・自動謝罪投稿（誤検知・荒らし悪用のリスク）
