# DENKEN-OS 自動化 実装指示インデックス

DENKEN-OS（電験二種 学習アプリ＋X発信）の運用で**自動化できる箇所をすべて洗い出し**、
それぞれの実装指示を保存したもの。**01〜12 の秘密情報不要な核ロジックは実装済み**
（`lib/` `scripts/` `.github/`、各 spec の「ステータス」を参照）。
04/08/12 は核ロジックのみ実装（OCR取込・Discord Bot 実体・Web Push 実配信は外部サービス/認証前提でアダプタ差込口）。

各ファイルはそのまま Claude Code に渡せるタスク仕様（ゴール／核心の設計判断／要件／
受け入れ条件／スコープ外／リスク）の形式。

## 横断原則（全自動化に共通）

1. **moat と安全は自動化しない** — 「会話/リプライ」「コミュニティの温度」「品質の最終確認」
   「訂正の承認」は人間が担う。自動化するのは下準備・配信・集計・通知。
2. **X凍結回避** — 正規API/公式スケジューラ経由のみ。投稿時刻・文面に**ゆらぎ**を入れ、
   連投しない。リプ/いいねの自動大量実行は禁止（即サスペンド要因）。
3. **著作権メタは全工程で必須** — 生成・取込・投稿のどの段でも出典/改題フラグを保持
   （`docs/x-strategy/04-compliance.md`）。
4. **秘密情報** — APIキーは env。`.gitignore` を確認し、絶対にコミットしない。
5. **人間が最終承認するゲート** — 公開（status=published）と訂正（retracted）は人手承認。

## 着手順（推奨）

```
1) 09 CI品質ゲート     … 先に品質の土台（PRの穴埋め・他の自動化の安全網）

   以降 04 / 07 / 08 / 10 / 11 / 12 を必要に応じて
```
