# 実装指示 11: クロスポスト／転載自動化

> ステータス: 実装済み（コア）。 実装: `lib/crosspost/repurpose.ts`。

## 0. ゴール
Xで作ったコンテンツ（今日の一問・図解・まとめ）を note / Instagram / YouTube Shorts へ
**媒体別に最適化して**展開し、1コンテンツの労力対効果を最大化する。

## 1. 核心の設計判断 ★（丸転載しない＝ベストプラクティス）
- **媒体ごとに形式を変える**（丸投げ転載は各媒体で伸びず、外部リンク扱いで不利になることも）。
  - X: 出題スレッド ／ note: 解説の長文記事 ／ IG: 図解10枚カルーセル ／ Shorts: 解法60秒動画
- 主戦場は X。転載は「資産化」目的（検索流入・別客層）。
- 各媒体のリンク/ハッシュタグ規約に沿って整形（重複リンクペナルティ回避）。

## 2. スタック
- n8n（README の自動化スタック）でワークフロー化。各媒体API or 下書きエクスポート。

## 3. 機能要件
- X投稿を起点に、媒体別フォーマットの**下書き**を自動生成
- note: 週次の「良問まとめ」を記事化（出典フッター付）
- Instagram: 図解をカルーセル画像に整形
- Shorts/YouTube: 解説台本＋字幕の下書き
- 投稿自体は媒体の公式機能/予約で（自動一斉投稿はしない＝凍結/規約回避）

## 4. 受け入れ条件
- [ ] 1つのXコンテンツから媒体別の最適化下書きが出る
- [ ] 各媒体で丸コピーになっていない（形式変換されている）
- [ ] 出典・リンク規約に沿っている

## 5. やらないこと
- 全媒体への同一文面の一斉自動投稿（伸びない＆規約リスク）
