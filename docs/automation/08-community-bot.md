# 実装指示 08: コミュニティBot（儀式の自動化）

> ステータス: 一部実装（核ロジック）。 実装: `lib/community/rituals.ts（儀式の純ロジック。実Botはdiscord.jsアダプタ）`。

## 0. ゴール
コミュニティ（Discord 想定）の**繰り返しの儀式**を自動化し、母集団の継続率を底上げする。
コミュニティ＝最強の堀（`06-moat-community.md`）。獲得コストが実質ゼロのチャネルに育てる。

## 1. 核心の設計判断 ★（関係性は人間／儀式は自動）
- **自動化するのは儀式**: チェックイン・もくもく会リマインド・出戻り歓迎・合格ロール付与。
- **自動化しないのは関係性**: 個別の励まし・質問への深い回答・トラブル対応は人間。
- Community-Led Growth: 成熟するとメンバー同士が24hでQ&A・問題解決を回す→質問は
  まずコミュニティに委ねる文化を Bot が後押し（自分が全部答えない）。

## 2. スタック
- Discord Bot（discord.js）。状態は軽量DB or Supabase。

## 3. 機能要件
- 朝: 「今日の予定」チェックイン投稿を促す／夜: 「結果」を促す
- 週次もくもく会の自動リマインド（固定枠）
- 久々ログインに「出戻り歓迎」メッセージ（罪悪感を与えない文面）
- 「今日の一問」連携（X出題をコミュニティにも展開）
- 合格報告に反応→「卒業生」ロール自動付与（社会的証明の可視化）
- 質問は専用チャンネルへ誘導し、回答者に感謝リアクション

## 4. 受け入れ条件
- [ ] チェックイン/リマインドが定時に投稿される
- [ ] 出戻りメッセージが前向きな文面
- [ ] 合格報告→卒業生ロール付与が動く
- [ ] Bot が個別の深い相談に立ち入らない（人間へエスカレーション）

## 5. リスク
- 通知過多はコミュニティ疲れ→離脱。頻度を絞り、ミュート可能に。
- Bot トークンは env。コミット禁止。

## 6. やらないこと
- 個別の励まし・相談対応の自動化（温度＝人間の仕事）
