# 実装指示 13: 試験問題バンクのベストプラクティス 100+（深堀調査・関連推奨）

> ステータス: 調査完了 + 高レバレッジ項目を本PRで実装。
> 目的: 電験(電気主任技術者試験)の問題を **どの学習サービスよりも充実** させるための、
> 学習科学・テスト理論(psychometrics)・既存サービス分析・電験ドメインに基づく実践集。
> 各項目に状態を付す: ✅=本PRで実装 / 🔜=スキーマ・基盤を用意（データ投入は継続） / 📋=推奨（将来）。

調査ソース要約（詳細は各セクション）:
- 試験区分・出題形式・科目合格制度（電気技術者試験センター, オーム社 schoolnavi, TAC, SAT, アガルート）
- 既存解説サービスの強み/弱み（電験王, 電験三種ドットコム, 電験合格(YouTube), 電気の神髄, 工事士.com）
- 出題傾向・頻出論点（電験王 科目別一覧, 電験三種まとめました, eleking, denken-muscle, 翔泳社「出るとこだけ」）
- テスト理論（item difficulty P / discrimination D / distractor efficiency DE, IRT, CTT; NCBI/PMC 各論文, ReCentering Psych Stats）
- 学習科学（spaced repetition / SM-2 / FSRS, retrieval practice, interleaving, worked examples, cognitive load）

## A. ドメイン正確性・電験固有（試験に忠実であること）

1. ✅ **全試験区分を表現**: 三種 / 二種一次 / 二種二次 / **一種一次・二次** を `exam` enum に含める（従来は一種が欠落）。
2. ✅ **科目モデルの忠実化**: 三種=理論/電力/機械/法規（5択マークシート）、二種二次=電力・管理/機械・制御（記述）を `subject`×`format`×`exam` で正しく拘束。
3. ✅ **記述式は自動採点しない**: `descriptive` は模範解答＋採点観点(`grading_points`)を提示し自己採点（二次の主戦場）。
4. ✅ **過去問の出典メタを構造化**: 年度(和暦/西暦)・回(上期/下期/一次/二次)・問番号を `exam_meta` で保持（電験王の「令和○年度 問○」表記に対応）。
5. ✅ **法令条文の明示**: 法規問題に根拠条文(`references`)を持たせる（電気事業法/電技/電技解釈/施行規則）。
6. ✅ **B問題（小問構成）対応**: 三種B問題は1問内に小問2つ→ `sub_questions` を表現できる構造。
7. 🔜 **科目合格制度の反映**: 学習計画で科目別の合格保有(3年/5回)を扱えるよう科目独立に集計。
8. ✅ **単位・記号の電験表記**: 回転速度は `min⁻¹`、電力 `kW`、`Ω`、`%Z` 等、業界・試験表記に統一。
9. ✅ **CBT対応を意識**: 5肢択一で順序非依存・図は代替テキスト必須（CBT/紙の両対応）。
10. ✅ **頻出論点を優先**: 理論=電気回路+電磁気、機械=四機(変圧器/直流機/誘導機/同期機)、電力=発変電+送配電、法規=電技+計算。被覆マップで優先度を明示。
11. ✅ **「四機」を厚く**: 機械の半分は四機。変圧器(効率/電圧変動率)・直流機(誘導起電力/速度)・誘導機(すべり/比例配分)・同期機(同期速度)を網羅。
12. ✅ **計算:論説の出題比を意識**: 理論8割計算、電力/法規は約半々、機械約半分が計算。calc/論説の両形式を用意。
13. 📋 **試験日・改定の追従**: 電技解釈の改正、CBT日程、科目構成変更に追従するデータ更新運用。
14. ✅ **頻出公式の明示**: 各問に使用公式(`formulas`)を列挙（暗記カード/逆引きに利用）。
15. 📋 **複数解法の提示**: 別解（例: 三相電力の `√3VIcosφ` と `3I²R`）を解説に併記（一部実装済）。

## B. スキーマ・データモデル（拡張性と一貫性）

16. ✅ **後方互換な拡張**: 追加フィールドはすべて optional。既存データ(T-0001..)を壊さない。
17. ✅ **zod ⇄ JSON Schema(ajv) のドリフト検知**: 両定義を同期し、テストで代表ケースを突合。
18. ✅ **誤答解説の永続化(`choice_explanations`)**: 各選択肢に「なぜ違うか/どの典型ミスか」を保存。**最大の学習資産**（従来は算出後に破棄していた）。
19. ✅ **タグ付け(`tags`)**: 横断検索・関連出題・弱点クラスタリングの基盤。
20. ✅ **学習目標(`learning_objectives`)**: 問題が測る到達目標を明示（逆向き設計）。
21. ✅ **前提知識(`prerequisites`)・関連論点(`related_topics`)**: 知識グラフ/推奨出題の基盤。
22. ✅ **段階的ヒント(`hints`)**: 答えを見る前の足場かけ（cognitive load 低減・self-explanation 促進）。
23. ✅ **想定解答時間(`estimated_time_sec`)**: ペース配分練習・模試時間設定に使用。
24. ✅ **数値採点の許容誤差(`numeric.tolerance`)と単位(`numeric.unit`)**: 有効数字差・丸め差を吸収。
25. ✅ **難易度1–5 + 実測補正**: 公称難易度に加え `stats.correct_rate` で再較正（item difficulty P）。
26. 🔜 **識別力(discrimination)指標**: 上位群/下位群の正答差を `stats` に拡張可能な設計。
27. 📋 **問題の版管理(version)・改訂履歴**: 誤り訂正(retracted→改訂)の追跡。
28. ✅ **ID 採番規約**: `T-xxxx`(検証済シード)/`G-xxxx`(生成)/科目接頭辞で衝突回避。
29. ✅ **status ライフサイクル**: draft→validated→published→retracted を厳格化（検証4項目）。
30. 📋 **多言語/ふりがな field**: 専門用語の読み仮名（学習初期の障壁低減）。

## C. 設問品質・選択肢設計（psychometrics）

31. ✅ **誤答は「成立する典型ミス」**: ランダム誤答禁止。力率二重掛け・√3忘れ等、実際に起こすミスを選択肢化。
32. ✅ **distractor efficiency**: 全誤答が機能する（誰も選ばない選択肢を作らない）よう、典型ミス由来に限定。
33. ✅ **選択肢の同質性**: 桁・単位・有効数字を揃え、正解が形式で浮かないようにする（昇順整列）。
34. ✅ **正解一意・有限・綺麗**: `clean_answer`（整数 or 小分母有理数）を強制。汚い値は破棄。
35. ✅ **answer ∈ choices をコードで担保**: JSON Schema で表現不能なためコード検証。
36. ✅ **選択肢の重複排除**: 正解と誤答、誤答同士の数値衝突を検出して破棄。
37. 📋 **「上記すべて/該当なし」回避**: 推測を誘発する選択肢を作らない方針を明文化。
38. ✅ **最頻誤答の記録(`common_wrong_choice`)**: 集計で「引っかかりやすさ」を可視化。
39. 📋 **選択肢数の一貫性**: 三種は5肢が基本（現行は4肢中心→5肢化を推奨）。
40. ✅ **物理的妥当性(`physically_valid`)**: 力率≤1、すべり∈(0,1)、効率≤1等を生成段で保証。
41. 📋 **手がかり語の排除**: 「常に/決して」等の絶対語が正誤の手がかりにならないよう論説問題を精査。
42. 📋 **設問文の独立性**: 前問の答えに依存しない（B問題の小問連鎖は明示）。

## D. ハルシネーション・正確性保証（本プロジェクトの moat）

43. ✅ **数値の正解はLLMに出させずコードで算出**: パラメトリック決定論テンプレート。
44. ✅ **解説の最終数値とコード正解の突合**: 不一致は破棄（`narrationMatchesAnswer`）。
45. ✅ **純関数ソルバで再現(`solver_checked`)**: テンプレートが真値を返す。
46. ✅ **人手検算ゲート(`human_checked`)**: 自動生成は draft 止まり、validated は人手確認必須。
47. ✅ **合格者監修(`supervisor_checked`)**: 重要論点は監修必須にできる任意フラグ＋`confidence`。
48. ✅ **テンプレートの単体テスト**: 既知 params→既知正解の固定テストで回帰防止。
49. ✅ **生成バッチの全件 validate**: N問生成→全件スキーマ＋不変条件通過をテスト。
50. 📋 **逆算検証(別解での再計算)**: 同一正解を独立な式で二重計算（一部実装）。
51. ✅ **citation 必須(改題)**: original 以外は出典必須をコード＋スキーマで二重化。
52. 📋 **既存解説の引き写し禁止**: 解説は自作生成のみ（コンプラ）。

## E. 被覆・網羅性（“どこより充実”の定量化）

53. ✅ **公式シラバスの機械可読化(`syllabus.ts`)**: 全科目×全区分の論点ツリーを定義。
54. ✅ **被覆レポート(`npm run coverage:syllabus`)**: 何%の論点がテンプレ/問題で埋まっているかを算出。
55. ✅ **被覆 CI ゲート**: 科目ごとの最低テンプレ数を下回ったら CI 失敗（退行防止）。
56. ✅ **論点優先度**: 頻出度(high/medium/low)をシラバスに付与し、high から埋める。
57. 🔜 **空白論点の可視化**: 未カバー論点を一覧出力（次に作るべき問題が分かる）。
58. 📋 **過去問頻度ヒートマップ**: 年度別出題回数で優先度を自動補正。
59. ✅ **区分別の最小集合**: 三種4科目すべてに複数論点テンプレを確保。
60. 📋 **二次記述の論点拡充**: 電力・管理/機械・制御の頻出記述テーマを継続追加。

## F. 学習科学・出題アルゴリズム（定着の最大化）

61. ✅ **間隔反復(SM-2/FSRS)**: 既存 `lib/scheduler` を活用、弱点 topic を優先。
62. ✅ **弱点診断→適応出題**: `tags`/`subject` 別正答率で次の一問を選ぶ。
63. 📋 **インターリービング**: 科目/論点を混ぜて出題（ブロック練習より定着）。
64. ✅ **想起練習(retrieval practice)**: 解答→即フィードバック→解説の順序を強制。
65. ✅ **ワークトイグザンプル**: `solution` を段階提示（認知負荷低減）。
66. ✅ **段階的ヒント**: いきなり答えでなく `hints` で足場かけ。
67. 📋 **自己説明プロンプト**: 「なぜこの式か」を問い返す UI（誤答解説と連動）。
68. 📋 **誤答からの学習ループ**: 誤答した typical-mistake を再出題スケジュールに反映。
69. 📋 **メタ認知の可視化**: 確信度申告→キャリブレーション表示。
70. ✅ **想定時間でのペース管理**: `estimated_time_sec` を模試/タイマーに利用。
71. 📋 **目標逆算（試験日カウントダウン）**: `lib/notify` の試験カウントダウンと学習量を連動。

## G. 解説・コンテンツの質

72. ✅ **誤答ごとの解説**: 「あなたが選んだ B は √3 を忘れた値」と個別フィードバック（`choice_explanations`）。
73. ✅ **公式の明示と意味付け**: `formulas` ＋解説で「丸暗記でなく意味」を促す（%Z等）。

80. 📋 **動画/外部良質解説への補助リンク**: 出典明記の上で補助参照（電験合格等）。
