# 実装指示 14: 試験問題の「品質」ベストプラクティス 100+（作問・心理測定）

> ステータス: 調査完了 + 高レバレッジ項目を実装。
> 13 が「充実（被覆・メタ）」なら、本 14 は **良問らしさ（item quality）** を扱う。
> 状態: ✅=本PRで実装 / 🔜=基盤あり・継続 / 📋=推奨（将来）。

調査ソース要約:
- **Haladyna, Downing & Rodriguez (2002)** 多肢選択式の項目作成31則（超定番ガイドライン）。
- **Rodriguez (2005)**: 機能する誤答は **3個前後が最適**。5択に水増しすると非機能誤答が増える。
- 非機能誤答(NFD)研究（受験者の<5%しか選ばない誤答は機能していない；5択は非機能誤答が増える）。
- NBME 項目作成マニュアル／BYU/UF の作問チェックリスト。
- 数値採点の許容誤差・有効数字（Pearson Mastering / WebAssign / Norton: 相対 2–3%・有効3桁が標準）。

**本プロジェクトの設計判断（重要）**: 研究に従い、**選択肢を無理に5つへ水増ししない**。
「機能する誤答（典型ミス由来）だけを置く」方針を採り、誤答妥当性・完全性・重複の少なさで品質を担保する。

## A. 設問（stem）の質

1. ✅ 1問につき測る概念は1つ（単一論点 = 1テンプレート1閉形式）。
2. ✅ 問いは設問文で完結（選択肢を読まずとも何を問うか分かる）。
3. ✅ 必要十分な数値・単位を明示（V/Ω/Hz/kW/%/min⁻¹ など電験表記）。
4. 📋 否定形（「正しくないものは」）は最小限・強調表示する。
5. ✅ 与件は現実的レンジ（realistic_range）に収める。
6. ✅ 物理的に成立する設定のみ（力率≤1、すべり∈(0,1)、効率<100%）。
7. 📋 図が要る論点（ブリッジ/分圧/回路）は figure/alt を付す（CBT対応）。
8. ✅ 曖昧さの排除（平衡条件など前提を明記）。
9. 📋 設問文の長さ・語彙を統一（読みやすさ）。
10. ✅ 単位を答に含めるか問題文で固定（numeric は unit を保持）。

## B. 選択肢（options）の質 — Haladyna 31則の中核

11. ✅ **誤答は妥当に（#29）**: ランダム誤答禁止。すべて「成立する操作」から生成。
12. ✅ **典型ミスから作る（#30）**: 力率二重掛け／√3忘れ／係数取り違え等、実際に起こす誤り。
13. ✅ 各誤答に「なぜ違うか」を保存（choice_explanations）。
14. ✅ **機能する誤答を3個前後（Rodriguez）**: 4択（1正+3誤）を既定とし、5択へ水増ししない。
15. ✅ 選択肢は重複なし（key の手がかり防止）。
16. ✅ 数値選択肢は昇順整列（位置で正解を推測させない）。
17. ✅ 正解は一意（answer ∈ choices をコード担保）。
18. ✅ 選択肢の同質性（桁・単位・有効数字を揃える）。
19. ✅ **荒唐無稽な誤答を作らない**: 例) 同期速度 28800 min⁻¹ のような非現実値を排除（テンプレ修正済）。
20. ✅ 誤答妥当性のサニティ判定（quality.ts: 符号反転・0・桁違い1000倍超を error、極端を warn）。
21. 📋 「上記すべて／該当なし」を使わない（推測誘発）。
22. 📋 文法・語尾を正解と誤答で揃える（手がかり排除）。
23. ✅ 最頻誤答（likelyWrongChoice / common_wrong_choice）を記録。
24. 📋 5択が必要な本試験準拠モードは将来オプション（既定は機能優先の4択）。

## C. 正解と検算（correctness）

25. ✅ **数値の正解はコードで算出**（LLMに出させない）。
26. ✅ 解説の最終値とコード正解の一致確認（不一致は破棄）。
27. ✅ 純関数で再現（solver_checked）＋人手検算ゲート（human_checked）。
28. ✅ 綺麗な値のみ採用（clean_answer。汚い draw は棄却）。
29. ✅ 別解での二重確認（例: 三相電力 √3VIcosφ ↔ 3I²R）。
30. ✅ 既知 params→既知正解の固定テスト（回帰防止）。
31. ✅ 改題は出典必須（citation）。

## D. 数値採点（numeric grading）

32. ✅ **相対許容誤差**（gradingTolerance: |answer|×1%、下限0.01）で丸め差を吸収。
33. ✅ 絶対値固定の過剰に厳格/緩い許容を撤廃（旧: 5880 に tolerance 0.1 等）。
34. ✅ 単位を numeric.unit に保持。
35. 📋 有効数字ベースの採点（標準: 有効3桁、中間計算は5桁保持）。
36. ✅ 許容が緩すぎ（相対>20%）を warn 検出。
37. 📋 別表記（分数・指数）受理の正規化。

## E. 重複・多様性（item pool quality）

38. ✅ **バッチ重複排除**（generate のベストエフォート de-dup）。同一params/answerの重複を回避。
39. ✅ 母集合を出し切ったら重複許容で件数を満たす（従来の件数挙動は維持）。
40. ✅ 重複検出を品質レポートに集計（subject|topic|statement|answer）。
41. ✅ 制約の強い論点は母集合を拡張（単相全波・クーロン等を拡充）。
42. 🔜 母集合サイズの可視化（小プールの論点に係数追加を促す）。
43. 📋 近似重複（数値が僅差）の検出。
