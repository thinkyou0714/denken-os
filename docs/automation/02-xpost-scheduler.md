# 実装指示 02: X投稿スケジューラ（今日の一問）

> ステータス: 指示のみ（未実装）。

## 0. ゴール
validated 問題から「朝出題／夜解答」の投稿テキストを生成し、最適時間に**予約投稿**する。
出典フッターを自動付与。**人間の手作業を、下書き生成と予約だけに削る**（持続性対策）。

## 1. 核心の設計判断 ★（X凍結回避＝根本原因対策）
2026のXは「正規API/公式機能経由か」が信頼スコアに直結し、
**同時刻に同内容を投稿し続けるのが最も凍結されやすい**。連投はスパム判定で即サスペンド。
- 投稿は **公式の予約投稿 or 正規 X API v2** のみ経由する。
- **時刻にゆらぎ**（±15〜30分のジッター）、**文面はテンプレを複数持って回す**。
- **リプライ・いいね・フォローは自動化しない**（エンゲージの核＝人間が担う／凍結要因）。
- 連投しない（1日の投稿は質を保てる範囲。`02-content-engine.md` の頻度に従う）。

## 3. スタック
- TypeScript + Node。X API v2（投稿）／または公式スケジューラへエクスポート。
- 入力: 01 エンジンの validated JSON。

## 4. 機能要件
- 朝: 出題テキスト生成（難易度★・選択肢・問い・ハッシュタグ3個以内・出典フッター）
- 夜: 同問の解答テキスト生成（03 の正答率を差し込み、朝投稿を引用する形）
- 予約: 受験者の活動帯（朝7時台/夜21時台）にジッター付きでスケジュール
- リンクは本文に貼らない（「詳細はプロフ」or 別リプ）
- バッチ: 週末に1週間分を一括生成→予約（`08-sustainability-kpi.md` のバッチSOP）

## 5. 受け入れ条件
- [ ] validated JSON 1件 → 朝/夜2つの投稿テキストを生成
- [ ] 文面テンプレが複数からランダム選択され、毎回同一にならない
- [ ] 予約時刻にジッターが入る
- [ ] 出典フッターが必ず付く（改題は citation 付き）
- [ ] 本文にURLを含めない検査が通る

## 6. リスク / コンプラ
- X API キー/トークンは env。コミット禁止。
- 自動“返信”機能は実装しない（凍結回避＋会話は人間の仕事）。
- API 無料プランはポスト数制限が厳しい→従量/上位プラン要否を検討。

## 7. やらないこと
- リプライ・DM・フォロー/いいねの自動操作
- 解答の正答率計算そのもの（→ 03 に分離）
