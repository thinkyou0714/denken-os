# 実装指示 12: リマインダ／通知

> ステータス: 指示のみ（未実装）。

## 0. ゴール
学習継続を後押しする通知（学習リマインド・ストリーク維持・試験カウントダウン・
夜解答の告知）を自動配信し、リテンションを上げる。

## 1. 核心の設計判断 ★（通知過多＝離脱の根本原因）
- 通知は**ユーザーが頻度を制御できる**こと（過剰通知は churn の最大要因）。
- 配信時刻に**ゆらぎ**を入れ、機械的な印象を避ける。
- 内容は前向き（罪悪感で追い込まない＝出戻り歓迎の思想）。

## 2. スタック
- アプリ: Web Push / PWA 通知。コミュニティ: LINE/Discord。
- スケジューラは Supabase + cron（or n8n）。

## 3. 機能要件
- 学習リマインド（ユーザー設定時刻 ± ジッター）
- ストリーク維持ナッジ（途切れそうなときだけ・控えめに）
- 試験カウントダウン（節目で）
- 「夜解答が出たよ」告知（今日の一問に参加した人へ）
- 通知設定画面（頻度/種類/オフ）必須

## 4. 受け入れ条件
- [ ] ユーザーが通知頻度・種類をオフにできる
- [ ] 配信時刻にゆらぎが入る
- [ ] 文面が前向き（プレッシャー過多でない）
- [ ] オプトアウトが即時反映される

## 5. やらないこと
- オプトアウト無視・高頻度のプッシュ（離脱・規約違反リスク）
