# 実装指示 07: KPI収集＆週次レビュー自動生成

> ステータス: 実装済み（コア）。 実装: `lib/analytics/weekly-review.ts`。

## 0. ゴール
X と アプリの指標を収集し、`08-sustainability-kpi.md` の週次レビュー雛形を**下書き自動生成**する。
人間は「検証して削る」判断に集中する。

## 1. 核心の設計判断 ★（小規模のノイズ対策＝ベストプラクティス）
- **追う指標は3つだけ**: 保存数 / リプ往復 / フォロー転換（バニティ指標は出さない）。
- 小規模では単発の数字はノイズ→**移動平均（4週）とトレンド**で見せる。過剰反応を防ぐ。
- 流入計測は **UTM/専用URL** ベース（どの投稿がアプリ流入を生んだか）。

## 2. 参照
- `docs/x-strategy/08-sustainability-kpi.md`（KPIと週次レビュー雛形）

## 3. スタック
- TypeScript + Node。X API（投稿メトリクス）／アプリ側は Supabase 集計。
- 出力: マークダウンの週次レビュー下書き（数値＋当たり/外れの候補）。

## 4. 機能要件
- 週次で 3KPI を集計（投稿別の保存/リプ/フォロー転換）
- TOP3 投稿・伸びなかった投稿を自動抽出
- 4週移動平均でトレンド表示
- アプリ流入（UTM別）と突き合わせ
- 「来週の重点1つ」の欄は空欄（人間が埋める）

## 5. 受け入れ条件
- [ ] 週次レビュー雛形が数値入りで生成される
- [ ] バニティ指標（インプ総数等）を主指標に出さない
- [ ] 移動平均が計算されている
- [ ] 判断（削る/増やす）は提案に留め、自動実行しない

## 6. やらないこと
- フォロワー数を主KPIに据える
- 数字に基づく投稿戦略の自動変更（人間の判断を残す）
