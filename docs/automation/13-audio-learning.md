# 実装指示 13: 音声学習（法規 聞き流し）— 特徴・ベストプラクティス 100

> ステータス: コア実装済み（`lib/audio/` ＋ `web/src/audio-player.ts` ＋ `web/src/browser-speaker.ts` ＋ `web/` UI）。
> 本書は音声学習アプリのベストプラクティスを100項目に整理し、各項目に推奨と実装状況を付す。
> 凡例: ✅=実装済み / ☑=推奨（軽量・将来着手） / ⏸=スコープ外（理由付き。多くはバックエンド/ネイティブ依存）

設計原則（不変）: **読み上げ対象は検証済みテキストのみ**。数値の正解はコードで算出済みのものを読むだけで、TTS に新情報を作らせない（ハルシネーション対策の維持）。音声合成は**ブラウザ Web Speech API**を既定とし、無料・オフラインで動かす。

---

## A. 読み上げ品質（発音・正規化）

1. ✅ 単位記号の読み変換（Ω→オーム, kW→キロワット, MVA→…）。`lib/audio/speech-text.ts`。
2. ✅ 〔単位〕表記の読み上げ（〔kW〕→「、キロワット」）。
3. ✅ 演算子・分数の読み（`/`→わる, `×`→かける, `=`→イコール）。
4. ✅ 平方根・指数（√→ルート, 8²→「8の2乗」, 10⁻⁶→「10のマイナス6乗」）。
5. ✅ 虚数単位 j の文脈読み（8+j6→「8たすジェイ6」）。
6. ✅ 変数名を壊さない（数字直後の単位のみ変換。V_p/R/P は温存）。
7. ✅ 専門用語の読み辞書（力率→りきりつ, 極数→きょくすう 等。OS音声差を吸収）。
8. ☑ 数値の桁読み最適化（"1800"→「せんはっぴゃく」固定読みの辞書化。TTSが概ね正しいので優先度低）。
9. ☑ 略語展開（B種→「ビーしゅ」, D種→「ディーしゅ」の明示読み）。
10. ⏸ SSML による細粒度の韻律制御（Web Speech API は SSML 非対応。クラウドTTS導入時に検討）。

## B. アクティブリコール（記憶科学）

11. ✅ 出題→**考える間（無音）**→正解の順（想起を促す）。`gap` セグメント。
12. ✅ 考える間の長さを可変（3/6/10秒）。`gapMs`。
13. ✅ 正解の2回読み上げ（暗記定着）。`repeatAnswer`。
14. ☑ 穴埋め(クローズ)音声（条文の数値をビープ/無音に置換して想起）。
15. ☑ 「正解を声に出してから聞く」プロンプト（産出効果）。
16. ☑ 直前問題の即時再テスト（テスト効果の活用、間隔ゼロ）。
17. ☑ 誤答の着眼も音声化（なぜ間違うかを聞かせる）。`solution` に既出のため拡張容易。
18. ⏸ 音声での自己採点入力（「正解/不正解と言ってください」音声認識）。Web Speech 認識はオフライン非対応端末が多く保留。
19. ☑ 1問を「問題のみ」「解説のみ」モードで分割再生。
20. ☑ キーワードの強調反復（重要語を2回読む）。

## 実装サマリ（本リポジトリ）

- 純ロジック: `lib/audio/speech-text.ts`（読み上げ正規化＋専門用語辞書）, `lib/audio/script.ts`（台本生成＋プレイリスト）。
- 再生制御: `web/src/audio-player.ts`（DOM非依存・`Speaker` 注入でテスト可能）。
- 音声合成: `web/src/browser-speaker.ts`（Web Speech API・DOM隔離）。
- UI: `web/index.html` + `web/src/app.ts`（再生群・字幕・設定永続化・ショートカット・MediaSession）。
- テスト: `tests/audio/speech-text.test.ts`, `tests/audio/script.test.ts`, `tests/web/audio-player.test.ts`。
