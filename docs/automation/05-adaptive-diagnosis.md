# 実装指示 05: 適応出題（弱点診断）

> ステータス: 実装済み（コア）。 実装: `lib/scheduler/`。README ロードマップ M3 に対応。アプリの核価値。

## 0. ゴール
ユーザーの解答履歴から弱点（論点）を診断し、弱点に合わせた問題を 01 エンジンで
無限生成して出題する。「弱点だけ無限に出る」をアプリ限定価値にする（課金理由）。

## 1. 核心の設計判断 ★（ベストプラクティス）
- 復習スケジューリングは **FSRS（Free Spaced Repetition Scheduler）** を採用。
  SM-2 の改良版で機械学習ベース、現行標準。**目標保持率（desired retention）**が最重要設定。
- 弱点は **論点(topic)単位の正答率＋FSRSの記憶安定度** の二層で判定。
- MVP は SM-2 ベースでも可（実装が軽い）。インターフェースを切って後で FSRS に差替え可能に。

## 2. 参照
- `docs/x-strategy/templates/problem-schema.json`（subject/topic/difficulty）
- 01 エンジン（弱点 topic を渡すと類題を生成）

## 3. スタック
- TypeScript + Supabase（解答履歴・記憶状態の保存）。
- スケジューラは純ロジックに分離（テスト可能に）。

## 4. 機能要件
- 解答ログ記録（problem_id, 正誤, 所要時間, topic, 日時）
- topic 別の習熟度＋次回復習日を FSRS/SM-2 で算出
- 「今日やるべき弱点 topic」を返す → 01 で該当 topic の問題を生成・出題
- 目標保持率をユーザー/デッキ単位で設定可能
- 二次(記述)は自己採点入力に対応（自動採点はしない）

## 5. 受け入れ条件
- [ ] 同 topic で連続不正解 → その topic の出題優先度が上がる
- [ ] 正解が続いた topic は復習間隔が延びる
- [ ] 目標保持率を変えるとスケジュールが変わる
- [ ] スケジューラ単体のユニットテストが緑

## 6. やらないこと
- 記述問題の自動採点（自己採点 or 監修者添削）
- 独自スケジューリングのゼロ発明（実績ある FSRS/SM-2 を使う）
