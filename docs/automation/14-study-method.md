# 実装指示 14: 合格のための学習ロジック（聞く→解く→弱点ループ）

> ステータス: コア実装済み（`lib/study/lesson.ts` ＋ `web/` レッスンUI）。
> 「合格するために必要な勉強法」をプロダクトのループとして埋め込むための設計。
> 数値の正しさはコード算出（既存原則）を維持し、本書は"学び方"の最適化に集中する。

---

## 0. ゴール（一文）
「ボタン一つで**聞いて→解いて→弱点が分かり→次に何をやればいいかが決まる**」一巡を、
学習科学（アクティブリコール・間隔反復・インターリービング）と電験の合格構造に沿って提供する。

## 1. 根拠（エビデンスに基づく勉強法）

| 原理 | 内容 | 本実装での織り込み |
|---|---|---|
| **テスト効果（想起練習）** | 読む/聞くより「思い出す」方が定着する | 聞き流しの直後に**同じ論点を出題**（`buildLesson`: listen と quiz は同集合） |
| **間隔反復(SRS)** | 忘れる直前に復習すると効率最大 | SM-2 の `dueMs` で復習対象を抽出（`dueTopics`・復習モード） |
| **インターリービング** | 同種を固めず混ぜると転移が上がる | プレイリストで同一 topic 連続を回避（`interleave`） |
| **二重符号化** | 音声＋文字で記憶痕跡を増やす | 聞き流し＋字幕＋原稿コピー |
| **アクティブな間（pause）** | 答えを想起する空白が定着を生む | 台本に「考える間」(`gap`)・正解の2回読み |
| **即時フィードバック** | 誤りは早く正すほど良い | 解答直後に正誤＋解説＋誤答の着眼 |
| **弱点集中（デリベレイト練習）** | 苦手に資源を寄せる | 弱点 topic 優先・間違い直しモード・科目別到達度 |
| **目標逆算（合格ライン）** | ゴールから今の不足を測る | 60%合格ラインとの差分（`summarizeLesson.toPass`） |

## 2. レッスンのループ（プロダクト化）

```
[開始ボタン]
   ↓ 聞く（インプット, n問。考える間つき）
[聞き終わり]
   ↓ 解く（アウトプット＝同じn問を想起テスト）
[採点]
   ↓ 集計（正答率・合格ライン到達・弱点 topic/科目）
[講評＋次アクション]  ──→  弱点モード/復習モードへ（間隔反復に接続）
```

- 「聞いた論点をそのまま解く」ことで、聞き流しを**受動から能動**へ変える。
- 結果は `LocalProgress`（SM-2）に記録され、次回の `dueTopics`/弱点に反映＝**自己強化ループ**。

## 3. 合格ロジック（電験の構造に合わせる）

- **科目合格ライン ≒ 60%**（`PASS_LINE = 0.6`）。レッスン正答率を 60% と比較し、不足問数を提示。
- **科目合格制**: 一度合格した科目は数年保留 → 弱点科目に集中投下するのが最適戦略。
  `passReadiness` が科目別到達度（onTrack/データ不足）を返し、資源配分を支援。
- **データ不足の扱い**: 試行が少ない科目は判定保留（`enoughData=false`）。早合点を避ける。

## 4. 実装（純ロジック・テスト可能）

- `lib/study/lesson.ts`
  - `buildLesson(problems, opts)` … 聞く列と解く列（同集合・別順序）を生成。
  - `summarizeLesson(results)` … 正答率・合格ライン到達・不足問数・topic/科目別・弱点。
  - `lessonFeedback(summary)` … 講評＋次アクションの自然文（画面表示＋読み上げ）。
  - `passReadiness(bySubject)` … 科目別の合格到達度（資源配分の指針）。
- `web/`: 「📚 レッスン（聞く→解く）」ボタン。聞き終わり(`onComplete.completed`)で出題フェーズへ。
  既存の採点 `grade()` に完了フック `onGraded` を追加し、レッスン中だけ次問へ送る。

## 6. 科目別合格到達度（永続・科目合格制の核）
- 解答ログ(`AnswerLog`)に `subject` を持たせ（後方互換: 旧ログは未設定）、`LocalProgress.subjectAccuracy()`
  でセッションを跨いだ科目別正答率を集計する。これを `passReadiness` に渡し、UI 上部に
  「✅合格圏 / ⚠️要強化 / ⏳データ不足」を科目別バッジで常時表示する。
- 科目合格制（合格科目は数年保留）に対し、⚠️要強化の科目へ学習資源を寄せる判断材料になる。
- 試行が少ない科目は ⏳（判定保留）にして早合点を避ける。

## 7. 合格逆算ペース（試験日からの日次目標）
- `lib/study/exam-plan.ts` の `planExam` が、試験日までの残り日数と科目別到達度から
  **今日やるべき問題数**と**重点配分科目**を逆算する（目標逆算＝合格戦略の核）。
- 弱点が大きい・残り日数が少ないほど `todayTarget` が増える。全科目合格圏なら習慣維持の最低目標。
- データ不足の科目は弱点扱いしない（早合点回避）。UI 上部に「あと◯日／今日◯問／重点科目」を表示。
- 試験日は `LocalProgress.setExamDate`/`examDateMs` で永続化。達成度は `todayAnswered` で可視化。
