# 実装指示 06: 学習記録シェア画像ジェネレータ

> ステータス: 指示のみ（未実装）。

## 0. ゴール
学習記録（連続日数・今日の学習時間・正答率推移）を、見映えする画像カードとして自動生成し、
ユーザーがXに投稿したくなる形で提供する。**投稿が毎回そのまま広告になるバイラルループ**。

## 1. 核心の設計判断 ★（成長の根本）
- 広告費ゼロの個人開発で**最も費用対効果が高い成長レバー**がバイラルループ。
  ストリーク/記録カードのシェアは Studyplus/Duolingo 型の実績ある仕組み。
- カードに**控えめなブランド要素**（ロゴ・アプリ名・#今日のDENKEN）を必ず入れる。
- ワンタップで X 投稿（テキスト雛形＋画像）まで繋げる（摩擦ゼロ）。

## 2. スタック
- Next.js の OG 画像生成（`@vercel/og` / Satori）or node-canvas。
- 入力: ユーザーの学習記録（05/Supabase）。

## 3. 機能要件
- カード種類: ①連続記録(ストリーク) ②今日の学習サマリ ③週間/月間振り返り
- デザインは回路図モチーフ＋カウントダウンで世界観統一（`01-positioning.md`）
- 投稿用テキスト雛形を同時生成（ハッシュタグ3個以内・リンクは本文に貼らない）
- 合格者の卒業カードは別テンプレ（社会的証明として強い）

## 4. 受け入れ条件
- [ ] 記録データからカード画像が生成される（3種類）
- [ ] カードにブランド要素が必ず入る
- [ ] 投稿テキスト雛形が同時に出る（URL本文貼り無し）
- [ ] 画像に個人情報（本名・メール等）が漏れない

## 5. リスク / プライバシー
- 共有画像に個人特定情報を載せない（表示名/任意のニックネームのみ）。
- 数値（学習時間等）の盛り・改ざんを防ぐ（記録ベースで生成）。

## 6. やらないこと
- 自動投稿（ユーザーの手動シェアが前提＝凍結回避＆主体性）
